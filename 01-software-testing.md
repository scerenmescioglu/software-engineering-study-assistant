# Software Testing Fundamentals

Software testing is the process of evaluating a software system to identify defects and determine whether it satisfies its specified requirements.

## Verification and Validation

Verification asks: "Are we building the product correctly?"

It checks whether software is being developed according to its specifications, design documents, and standards.

Validation asks: "Are we building the correct product?"

It checks whether the completed software satisfies the needs and expectations of its users.

## Testing Levels

### Unit Testing

Unit testing evaluates individual functions, methods, classes, or modules in isolation.

It is commonly performed by developers during implementation.

### Integration Testing

Integration testing evaluates whether multiple components or modules work correctly together.

It focuses on interfaces, data exchange, and communication between components.

### System Testing

System testing evaluates the complete integrated system.

It checks the system against its functional and non-functional requirements.

### Acceptance Testing

Acceptance testing determines whether the software is ready to be accepted by the customer or end user.

It may include alpha testing, beta testing, and user acceptance testing.

## Black-Box Testing

Black-box testing examines the externally visible behaviour of software without considering its internal code.

Common black-box techniques include:

- Equivalence partitioning
- Boundary value analysis
- Decision table testing
- State transition testing
- Use-case testing

## White-Box Testing

White-box testing examines the internal structure and logic of the software.

Common white-box coverage criteria include:

- Statement coverage
- Branch coverage
- Condition coverage
- Path coverage

## Regression Testing

Regression testing checks that existing functionality still works after software has been changed.

It is performed after bug fixes, updates, refactoring, or the addition of new features.

## Functional and Non-Functional Testing

Functional testing checks what the system does.

Examples include login, registration, calculations, search, and payment operations.

Non-functional testing checks how well the system operates.

Examples include:

- Performance testing
- Security testing
- Usability testing
- Compatibility testing
- Reliability testing
- Load and stress testing

## Testing Principle

Testing can reveal the presence of defects, but it cannot prove that a software system contains no defects.
